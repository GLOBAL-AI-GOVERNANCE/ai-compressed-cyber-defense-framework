# Contributing

Thank you for helping improve AI-Compressed Cyber Defense Framework.

This repository accepts defensive, public-safe contributions only.

## Contribution Rules

Contributions should:

- Improve defensive architecture, systems security engineering, mission resilience, or evidence-based verification.
- Preserve public-safe boundaries.
- Cite sources for factual claims.
- Avoid exploit code, target-specific attack procedures, credential-abuse instructions, or operational red-team playbooks.
- Use bounded language for AI capability claims.

## Preferred Contribution Types

- Clearer checklists
- Better evidence templates
- Sanitized examples
- NIST SP 800-160 mappings
- Defensive implementation notes
- Public-safe source updates

## Review Criteria

A contribution should answer:

1. Is it defensive?
2. Is it source-grounded?
3. Is it useful to a practitioner?
4. Does it avoid overstated claims?
5. Does it improve verification or usability?
